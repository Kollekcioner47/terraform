Давайте создадим ресурс по файлу lesson2.tf
Дальше мы будем вносить изменения в ресурс, 
Давайте добавим ядер и памяти нашему серверу
изменим блок resources на:

resources {
    cores  = 4
    memory = 4
  }

проверяем и запускаем
terraform plan
terraform apply

и мы получаем отказ, поскольку яндекс запрещает менять работающие машины 
по-умолчанию

необходимо в tf файл добавить :
под полем name добавляем
allow_stopping_for_update = true

#! во время изменения сервис будет остановлен !

Попробуйте добавить к своему серверу labels:
labels = {
    name ="mysuperserver"
    }

Попробуйте прописать лейбл вручную, через яндекс консоль, а затем снова
запустите terraform apply, что произошло?

проверяем в консоли яндекса наши ресурсы, после не забываем всё уничтожить командой 
terraform destroy

второй урок законен
